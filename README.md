# IC Imagine · NC Math 1 — Class Website

A complete, self-contained class site: a homepage, a Class Info hub, and **all 8 NC Math 1 units**.
Each unit has three pages — a **landing page**, an interactive **Guided Notes** packet
(flowchart-driven, click-to-reveal), and a self-checking **Practice** worksheet. Plain
HTML/CSS/JS, no build tools, ready for GitHub Pages or any static host.

## What's inside

| File | What it is |
|------|------------|
| `index.html` | Homepage — hero + grid of all 8 units. |
| `class-information.html` | Syllabus/pacing/policy hub + year-at-a-glance + auto-activating download cards. |
| `unitN.html` | Unit landing page: learning targets, vocabulary, links to Notes & Practice. |
| `unitN-notes.html` | Interactive guided notes: worked examples, flowcharts, SVG graphs, fill-in blanks. |
| `unitN-practice.html` | Self-checking practice with a live score. |
| `theme.css` | The whole design system. Edit the `:root` colors to re-theme every page at once. |
| `script.js` | Shared interactions (reveal, grader, matching, score). You normally won't touch this. |

## The printable PDF for each unit

Every Notes and Practice page is print-optimized, so **one page gives you two PDFs**:

- **Blank student worksheet** — just **Print → Save as PDF**. Answers, worked-example
  solutions, and feedback are hidden; blanks print as write-in lines.
- **Answer key** — click **"👁 Reveal all answers"** at the top first, *then* Print → Save as PDF.

The nav bars, buttons, and score box are automatically hidden when printing.

## How the interactive pieces work

- **Fill-in blank:** `<span class="blank" data-ans="answer" onclick="rb(this)">?</span>`
- **Worked example step:** inside `<div class="steps" data-example="id">`, each
  `<div class="step" onclick="reveal(this)">label<span class="placeholder"> click to reveal</span><span class="hidden-content"> answer</span></div>`.
- **Multiple choice:** `<button class="chk-btn" onclick="chk('qid','CORRECT TEXT','THIS OPTION')">…</button>`
  (the 2nd argument is the correct text, the 3rd is this button's text).
- **Typed check:** `chkVal('qid', ['accepted','answers'])` — answers are matched loosely
  (spaces, case, − vs -, ≤ vs <= all normalized).
- **Class Info cards:** each has `data-file="name.pdf"`; drop a PDF with that exact name in
  the folder and the card becomes a live download automatically.

## Editing content (optional)

The pages are generated by a small Python toolkit you can keep or discard:
`gen_lib.py` (page chrome + SVG visual generators), `build.py` (shells + homepage),
`content.py` (the authored math content for every unit). Re-run `python3 content.py`
to regenerate all pages. To hand-edit, just open any `.html` file directly — the markup
is readable and uses the patterns above.

## Publish on GitHub Pages

1. Create a repo and upload every file in this folder.
2. **Settings → Pages → Build from branch → main → /(root)** → Save.
3. Live at `https://<your-username>.github.io/<repo-name>/`.
